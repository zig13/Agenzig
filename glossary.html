Agenzig Code Glossary

PC = Player Character

Effects

  • Change to character condition either temporarily or permanently

    • A temporary effect may come from having an item present in the inventory, a piece of equipment equipped or possessing an ability

      • This would included being granted the combat action "Stab" while having a dagger equipped

      • Temporary effects to not directly edit the character file

      • The effects of items, equipment and abilities are applied when the character is loaded

    • A permanent effect may arise from a choice, action or activation

      • This would include changing the scene or adding an item to the PC inventory

      • Permanent effects directly alter the character file


Action

  • An option available to the player

  • Have permanent effects

  • Available by default in all scenes (not available in confrontations) but can be blocked on a scene-by-scene basis

  • Can have requirements exactly like choices

  • Spells would be put into an adventure as actions that require a certain minimum amount of a currency – mana – to be available and have the effect of reducing total mana by that amount in addition to their beneficial effect


Choice

  • An action granted by the PC’s presence in a scene(state)

  • May be subject to requirements

  • The same choice could be present in multiple scenes but it’s availability must be implicitly stated each time

  • A choice will always have the same outcome (i.e. effects) regardless of the scene or the PC’s Attributes, Vitals etc

    • A temporary effect may come from having an item present in the inventory, a piece of equipment equipped or possessing an ability

      • This would included being granted the combat action "Stab" while having a dagger equipped

      • Temporary effects to not directly edit the character file

      • The effects of items, equipment and abilities are applied when the character is loaded

    • A permanent effect may arise from a choice, action or activation

      • This would include changing the scene or adding an item to the PC inventory

      • Permanent effects directly alter the character file


Scene

  • Scenes are used to present a situation to the player for them to resolve using choices and actions

  • The vast majority of adventures will use Scenes but they are defined as “optional” as it is possible for an adventure to use only Confrontations


Scene State

  • Allows the PC to have a permanent impact on the world

  • The simplest example is that if the PC were to pick a potion up off the floor then the description of the room would no longer mention the potion and the choice to pick up the potion would no longer be available

  • Can also be used to keep track of NPC conversations


Ability

  • An alternative way of imparting temporary effects

  • Work very similarly to items and equipment except a lack of slots and interactions

  • Makes it easier to add/remove a group of effects on the PC

  • An ability called supper-strength could boost strength by 2, provide a combat action – super punch, provide a regular action (flex biceps) and could be checked for by the requirement of a choice.


Item

  • An entry in the inventory database

  • It’s presence may be checked for by a requirement

  • While an item is in the inventory it may effect the PC

  • The PC can carry any amount of items simultaneously


Activation

  • An action in the specific context of a specific item or equipment

  • Put another way it is an action with the parameter – the item

  • Allows the PC to interact with items in their possession

  • An activation command word (e.g. equip, unequip, drop, use, eat, drink) can be used by any number of items and/or equipment but the effect it has is specified individually for each item in items.agez

Encounter Actions

  • Actions only available during confrontations

  • Can have all the effects and requirements that actions have but can also effect the attributes, vitals and currency of the encounter

  • All actions also have a time element which represents on a relative scale how long it takes to perform the action (hence why standard actions cannot be used in a confrontation)

  • Might include an interruption system?


Attributes

  • Arbitrary values

  • Unlike currencies, the actual numerical attribute value is hidden from the player and instead they are presented with qualatative descriptions of the values

  • Post commonly used to represent prowess or lack thereof of the PC and confrontations.

  • Can be used as a requirement for choices and actions

  • Can be referenced in confrontation actions


Vitals

  • Arbitrary values

  • Like attributes, vitals can be used as a requirement

  • If Attributes, Vitals can be referenced and their numerical values are hidden

  • If any PC Vital reaches 0, then the PC fails/dies/loses. The character file is deleted and the game exited.

  • Success in a confrontation requires the reduction of at least one of the confrontation’s Vitals to 0
